【操作系统原理】 Lecture 6 物理内存管理：非连续内存分配

发表于2025-11-05|更新于2025-12-08|技术学习操作系统原理
|浏览量:

一、非连续内存分配的需求背景

设计目标

  1. 解决连续分配的缺陷：
    • 外部碎片：内存中存在大量不连续的小空闲块，难以利用。
    • 内部碎片：分配的块大于实际需求（如伙伴系统）。
  2. 支持灵活的内存布局：
    • 允许程序代码、数据、堆栈等分散在物理内存的不同位置。
    • 便于实现内存共享和保护（如共享库）。

实现方式

  • 段式管理：按逻辑模块划分内存（如代码段、数据段）。
  • 页式管理：将内存划分为固定大小的页，按需分配。
  • 段页式管理：结合段式逻辑划分与页式物理分配。

二、段式存储管理

段地址空间

  1. 逻辑结构：
    • 进程地址空间由多个段组成，如：
      • 主代码段、子模块代码段、堆栈段、堆数据段。
    • 二维地址：(段号, 段内偏移)。
  2. 非连续性：
    • 各段可独立加载到物理内存的任意位置，无需连续。
      ### 段访问机制
  3. 硬件实现：
    • 逻辑地址：(段选择子, 偏移量)。
    • 段表（GDT/LDT）：存储段基址和段限长。
    • 地址转换：
      • 物理地址 = 段基址 + 偏移量。
      • 检查偏移量是否超过段限长，否则触发异常。
  4. 优点：
    • 支持内存共享（如多个进程共享代码段）。
    • 便于模块化编程和保护。
  5. 缺点：
    • 外部碎片问题仍存在（段长度可变）。
    • 段表查找效率较低。

三、页式存储管理

核心概念

  1. 页（Page）：逻辑地址空间划分为固定大小的块（如4KB）。
  2. 帧（Frame）：物理内存划分为与页大小相同的块。
  3. 地址表示：
  • 逻辑地址：(页号, 页内偏移)。
  • 物理地址：(帧号, 帧内偏移)。
    ### 地址转换流程
  1. 逻辑地址分解：
    • 页号 p = 逻辑地址 / 页大小。
    • 页内偏移 o = 逻辑地址 % 页大小。
  2. 页表查询：
    • 通过页号 p 查询页表，获取对应帧号 f。
  3. 物理地址生成：
    • 物理地址 = f * 页大小 + o。
      ### 优缺点
  4. 优点：
  • 无外部碎片（仅可能有少量内部碎片）。
  • 支持虚拟内存（按需加载页）。
  1. 缺点：
  • 页表占用内存大（每个进程需维护一个页表）。
  • 地址转换需访问内存两次（一次查页表，一次访存），性能低。

四、页表优化技术

快表（TLB, Translation Lookaside Buffer）

  1. 作用：缓存近期访问的页表项（页号 → 帧号映射）。
  2. 工作流程：
    • 首先查询TLB，若命中则直接获得帧号。
    • 若未命中，查询内存中的页表，并更新TLB。
  3. 效果：显著减少地址转换时间（TLB命中时仅需1次访存）。
    ### 多级页表
  4. 问题：大地址空间（如64位）导致单级页表占用内存过大。
  5. 解决方案：
    • 二级页表：
      • 一级页表：存储二级页表的基址。
      • 二级页表：存储实际的页表项。
    • 地址转换：
      • 一级索引 = 页号的高位部分。
      • 二级索引 = 页号的低位部分。
  6. 优点：
    • 仅需为实际使用的页分配二级页表，节省内存。
      ### 反置页表
  7. 设计目标：解决大地址空间下页表过大的问题。
  8. 结构：
    • 全局唯一页表：以物理帧为索引，存储 (进程PID, 页号)。
    • Hash表：通过 (PID, 页号) 计算Hash值，快速定位帧号。
  • 优点：
    页表大小固定（与物理内存帧数相关）。
  • 缺点：
    Hash冲突需额外处理，增加复杂度。

五、段页式存储管理

设计需求

  • 结合段式与页式优点：
    • 段式：支持逻辑模块划分和共享。
    • 页式：解决外部碎片问题。
      ### 实现机制
  1. 地址分解：
    • 逻辑地址 → (段号, 段内偏移)。
    • 段内偏移 → (页号, 页内偏移)。
  2. 地址转换：
    • 段表：查询段基址（页表基址）。
    • 页表：查询帧号。
    • 物理地址 = 帧号 * 页大小 + 页内偏移。
      ### 内存共享
  • 共享段：多个进程可共享同一段（如代码段）。
  • 共享页：通过页表映射到同一物理帧。
    ### 优缺点
  • 优点：
    • 兼具段式逻辑灵活性和页式物理高效性。
  • 缺点：
    • 地址转换需两次查表（段表 + 页表），性能开销大。
