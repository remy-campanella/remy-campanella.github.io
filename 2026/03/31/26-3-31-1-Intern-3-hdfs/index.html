【实习实训】 第三节 Hadoop 分布式文件系统（HDFS）

发表于2026-03-31|更新于2026-03-31|技术学习实习实训
|浏览量:

HDFS 的功能

HDFS 是 Hadoop 的核心项目之一，是分布式计算中数据存储管理的基础，基于数据流模式访问和处理超大文件，可以运行在廉价的商业服务器上。

HDFS 的定义

随着数据量越来越大，单个操作系统存不下所有数据，需要分配到更多操作系统管理的磁盘中。HDFS 是一个文件系统，通过目录树定位文件；它是分布式的，由多台服务器组成的集群联合起来实现其功能。HDFS 适合一次写入，多次读出的场景，不支持文件修改，适用于数据分析而不是网盘。

HDFS 的特点

HDFS 具有高容错性、高可靠性、高可维护性、高可扩展性、高获得性、高吞吐率等优点。数据自动保存为多个副本，当一个副本丢失，可以通过其他副本恢复。

缺点是不适合低延迟访问，处理大量小文件效率较低（大量小文件大量占用 NameNode 内存，小文件寻址时间也会超过读取时间，违反 HDFS 设计初衷），不支持并发写入与修改，仅支持数据追加而不是随机修改。

HDFS 的架构

HDFS 架构主要由 HDFS Client、Namenode、DataNode 和 Secondary Node 组成。

NameNode

NameNode 管理 HDFS 的命名空间，配置副本策略，管理数据块映射，处理客户端读写请求。

DataNode

即 Slave。执行来自 NameNode 的命令。存储实际的数据块，执行数据块读写操作。

Client

客户端，提供用户交互界面。

  • 文件切分，上传文件时将文件切分为数据块。
  • 与 NamenNode 交互，获取文件位置。
  • 与 DataNode 交互，读写数据。
  • 提供命令以管理 HDFS。

Secondary NameNode

Secondary NameNode 不是 NameNode 的热备份。它辅助 NameNode，分担其工作量，如周期性合并 Fsimage 和 Edits。在紧急情况下可辅助恢复 NameNode。

HDFS Shell

HDFS 的命令行终端程序一般位于 $HADOOP_HOME/bin/，命令为 hadoop fs 或 hadoop dfs。所有的 FS Shell 命令使用 URI 路径作为参数。大多数 FS Shell 命令的行为和对应的 Linux Shell 命令类似。

URI 格式是 scheme://authority/path。HDFS 的 scheme 是 hdfs，对本地文件系统，scheme 是 file，如果直接写路径则默认为配置指定（一般为 hdfs）。authority 即服务器地址 + 端口，例如 hadoop001:9000。
