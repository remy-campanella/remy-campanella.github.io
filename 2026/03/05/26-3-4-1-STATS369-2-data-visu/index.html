Why Data Visualisation Matters

  • First impression – visuals are immediately engaging
  • Efficient exploration – always visualise data before modelling
  • Effective communication – “a picture is worth a thousand words”
  • Summary statistics alone can be misleading (see Anscombe’s quartet)

The {ggplot2} Package

  • Based on The Grammar of Graphics (Leland Wilkinson, 2005)
  • Developed by Hadley Wickham (2007)
  • A layered grammar for building plots step by step

“A statistical graphic is a mapping from data to aesthetic attributes (colour, shape, size) of geometric objects (points, lines, bars).” – Hadley Wickham

Core Components

Component Description
Data The dataset you want to visualise
Aesthetics (aes) Map variables to visual properties (x, y, colour, size)
Geometric objects (geoms) The type of plot (points, bars, lines, etc.)
Facets Subplots by categorical variables
Stats Statistical transformations (e.g., binning, smoothing)
Coordinates Coordinate system (e.g., Cartesian)
Theme Non-data ink (labels, background, grid)

Which Plot to Use?

One Variable (Univariate)

Data type Recommended plot What to look for
Quantitative Histogram, density plot, boxplot Shape, peaks, centre, spread, outliers
Qualitative Bar plot, pie chart Majority/minority groups, gaps

Two Variables (Bivariate)

Combination Recommended plot What to look for
Quant vs Qual Side-by-side boxplots/densities Compare distributions across groups
Quant vs Quant Scatter plot, line plot Correlation, trend, outliers, seasonality
Qual vs Qual Stacked bar plot, mosaic plot Group proportions, dependencies

For 2+ variables, reduce to conditional bivariate plots using facet or colour.

General Design Tips

  • Use colours and shapes sparingly – don’t overcomplicate
  • Keep a clear message – every plot should tell one story
  • Pay attention to scales, labels, and hierarchy
  • Leave white space – don’t crowd the plot
  • Always check: are the axes labelled? Is the legend clear?
