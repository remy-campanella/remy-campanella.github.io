Example 1: Robot in Grid World

A robot is navigating a room. The space in which the robot can move is depicted as a “grid” (turning a continuous space into a discrete set
of coordinates).

The robot starts at location \(S\) and must reach location \(G\). The robot can move forward, backward, turn left or right, but cannot pass through walls or obstacles.

Suppose the robot knows the layout of the room. How can the robot perform its task?

Question: How can we model the task abstractly?

Representation is a fundamental AI task: choosing what information the algorithm will be allowed to reason about.

Formulating AI problems as states, actions, transitions, goal tests, and path costs.

The Search Problem

Search Problem
A search problem is specified by a state representation, an initial state, actions, a transition model, a goal test, and a path-cost function.

State

State
A state is an abstraction of the relevant information needed to describe the current situation for the purpose of solving the problem.

A state is only an abstraction of the world. It encodes the information that the model needs.

Transition Model

Transcition Model
The transition model specifies the state that results from taking an action in a state
\[ \mathsf{Result}(s, a) = s' \]

Goal Test

Goal Test
The goal test checks whether a state is successful.
\[ \mathsf{GoalTest}(s) \in \{\text{True}, \text{False}\} \]

Path Cost

Path Cost
The path cost measures the total cost of an action sequence.
\[ \sum_i \mathsf{StepCost}(s_i,a_i, s_{i + 1}) \]

Do not define the state, goal, or cost before understanding what problem the agent is actually solving.

Example 2: Route Planning

A route planner can also be viewed as state-space search problem. The city consists of roads and intersections.

A user using the route planner starts at a location, and hoping to reach a destination location.

The user may traverse on specified roads.

The cost is measured by time, distance, toll, fuel, or risk, etc.

A route planner becomes a precise problem only after we choose the state variables and cost function.

Example 3: Water-Jug Problem

You have a 3-litre jug and a 5-litre jug. Initially both are empty. You may fill a jug, empty a jug, or pour from one jug into the other. The goal is to obtain exactly 4 litres in the 5-litre jug.

Component Formulation
State \((x, y)\), \(x\) is the amount of the 3-litre jug and \(y\) isthe amount of 5-litre jug. \(0 < x < 3\), \(0 < y < 5\).
Initial State \((x, y) = (0, 0)\).
Actions Fill 3L, fill 5L, empty 3L, empty 5L, pour 3L to 5L, pour 5L to 3L.
Transcition Model Update water amounts according to the selected action.
Goal Test \(y = 4\).
Path Cost \(1\) per action by default.

State Sufficiency: The Key Test

Sufficiency
A state representation is sufficient if it contains enough information to determine legal actions, successor states, goal status, and path costs.

Example 6: Missionaries and Cannibals

Three missionaries and three cannibals must cross a river using a boat that carries at most two people. On either bank, if missionaries are present, they must not be outnumbered by cannibals.

State Space

State Space
Once the model is defined, the state space graph is the graph of all reachable states and transitions.
  • Vertices are states.
  • Edges are transitions by actions.
  • A solution is a path from the initial state to goal state.

State Space Size

Adding each variable can multiply the number of states.

Suppose a grid has \(N\) reachable cells.

Good representation is sufficient, but not unnecessarily large.

Search Node

Search Node
A node stores a state plus information about how the search reached it: parent node, action, depth, and path cost.

A node tells us how the search got there.

Modelling Workflow

  • What is the objective?
  • What information affects future actions and costs?
  • What actions are legal in each state?
  • What does each action change?
  • What is the goal test?
  • What is the path cost?

Quality Criteria for State Representations

Property Meaning
Sufficient Contains enough information to determine legal actions, costs, successor states, and goal status.
Compact Avoids irrelevant details that unnecessarily enlarge the state space.
Computable Allows successor states and costs to be generated efficiently.
Appropriate Matches the objective, constraints, and cost function of the actual problem.
