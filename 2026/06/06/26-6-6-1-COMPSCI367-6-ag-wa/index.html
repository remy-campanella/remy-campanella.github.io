【COMPSCI 367】 Lecture 6 A* Guarantees and Weighted A*

发表于2026-06-06|更新于2026-06-11|技术学习COMPSCI 367 Artificial Intelligence

Recall: A* Algorithm

The performance of A* algorithm greatly relies on the heuristic function h. So what \(h(n)\) should we design?

  • Admissibility: Optimal solution guarantee
  • Informedness: More useful heuristic
  • Consistency: Efficient implementation guarantee
  • Weighted A*: Making A* more greedy

Admissibility: Optimal Heuristic

A heuristic h is admissible if it never overestimates the true cheapest remaining cost.
\[ h(n) \leq h^*(n), \text{for every node \ } n \]

Symbols Type Known?
\(h(n)\) Heuristic estimate Yes.
\(h^*(n)\) True cheapest remaining cost Usually no.
\(C^*\) Optimal solution cost from the start Usually no until the search succeeds.

Admissibility allows heuristic to be:

  • too small.
  • weak.
  • zero.

Admissibility forbids heuristic to be:

  • exaggerate the cost left.
  • tule out a path being too pessimistic.

To prove admissibility, ask: is this estimate always a lower bound on the true cheapest remaining cost?

Optimality of A*
Let
\[ C^* = \text{cost of an optimal solution path from the start to a goal} \]

Goal of the proof
Show that A* cannot remove a suboptimal goal while there is still a frontier node on an optimal path with smaller or equal priority.

Suppose that \(n\) lies on an optimal path. Then thecost already paid plus the true remaining cost is the optimal solution cost
\[ g(n) + h^*(n) = C^* \]

By admissibility, wederive the core inequality
\[ \begin{align} f(n) &= g(n) + h(n) \\ &\leq g(n) + h^*(n) \\ &= C^* \end{align} \]

For every frontier node \(n\) that lies on an optimal path, \(f(n) ≤ C^*\).

Why A* pick a worse goal first?
Suppose A* is about to remove a suboptimal goal \(G_{bad}\) with
\[ g(G_{bad}) > C^* \]

For a goal node, \(h(G_{bad}) = 0\), so
\[ f(G_{bad}) = g(G_{bad}) > C^* \]

But before the optimal path is completed, there is some frontire node \(n\) on an optimal path with
\[ f(n) \leq C^* \\ f(n) < f(G_{bad}) \]

A* would remove \(n\) before \(G_{bad}\). A*cannot return a worse goal first.

Informedness: More Useful Heuristics

Suppose \(h_1\) and \(h_2\) are both admissible. We say \(h_2\) is more informed than \(h_1\) if
\[ h_2(n) ≥ h_1(n) \]

for all non-goal nodes \(n\), with strict inequality somewhere.

Consistency: Efficient Implementation Guarantee

A heuristic is consistent if every one-step move satisfies a triangle-inequality-like condition.
\[ h(n) \leq c(n, n') + h(n') \]

for every edge \(n \to n'\).

The heuristic should not drop by morethan the cost of the edge just taken.

Why Consistency Matters for Graph Search
Once A* removes a state from the priority queue, the cheapest path to that state has already been found.

Consistency is the condition that makes A* graph search behave like Dijkstra’s algorithm with heuristic guidance.

Weighted A: Making A More Greedy

Weighted A* changes the priority function by giving the heuristic extra weigh
\[ f_w(n) = g(n) + wh(n), w > 1 \]

Pseudocode

Weighted A* deliberately breaks safe optimism to search more aggressively.

Practical use
Weighted A* is useful when a good solution quickly is more valuable than a perfectly optimal solution too late.

Some analyses of weighted A* show a bounded-suboptimality guarantee.

A search strategy is \(\varepsilon\)-admissible if it guarantees a solution with cost at most
\[ (1 + \varepsilon)C^* \]

For weighted A* with
\[ f_w(n) = g(n) + wh(n), w > 1 \]

under the usual assumptions for the weighted A* bound, one can guarantee
\[ \text{returned solution cost} \leq wC^* \]

then weighted A* is
\[ (w - 1)\text{-admissible} \]
