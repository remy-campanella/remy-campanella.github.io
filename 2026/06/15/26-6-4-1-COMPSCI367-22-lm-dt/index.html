Weighted Evidence and Hierarchical Rules

Logistic regression learns weighted evidence; decision trees learn hierarchical rules.

From Features to Evidence

Running Example: Spam
Positive class: \(y = 1\) means spam. Negative class: \(y = 0\) means not spam.

A feature can provide evidence toward or away from the positive class.

Logistic Regression

Linear Score
A linear score is a weighted sum of input features plus a bias.
Weight
A weight controls how strongly a feature contributes to the score.
Bias
A bias is a constant term that shifts the score before any features are considered.

Linear Score
\[ s = b + w_1x_1 + w_2x_2 + \ldots + w_nx_n \]

Interpreting Weights

Weight Type Intuition, all else equal
positive Increasing this feature increases the score of spam.
negative Increasing this feature decreases the score of spam.
large magnitude Stronger direct effect on the score.
near zero Weak direct effect on the score.

Common misconception
A positive weight does not mean the feature is “good”. It means the feature increases the score for the positive class.

From Score to Probability

The logistic function maps any real-valued score to a number between 0 and 1(\((0, 1)\)).

\[ \sigma(x) = \frac{1}{1 + e^{-s}} \]

Previous score
\[ \sigma(2.5) = \frac{1}{1 + e^{-2.5}} \approx 0.924 \]

The model estimates about 92.4% probability of spam. With a 0.5 threshold, it predicts spam.

The score can be any real number; the logistic function converts it into a probability like output.

Linear Decsion Boundaries

Decision Boundary
A decision boundary is the set of points where a classifier changes its predicted class.

Two-feature example
\[ s = -3 + 0.8x_1 + 1.2x_2 \]

The boundary is
\[ -3 + 0.8x_1 + 1.2x_2 = 0 \]

which is a straight line in two-dimensional feature space.

Data pattern Logistic regression with original features
Classes roughly separated by a line Natural fit.
One class forms a ring or local island Difficult without new features.

A linear model can become more expressive if we engineer useful features.

Original information Possible engineered feature
\(x_1\), \(x_2\) \(x_1x_2\), \(x_1^2, x_2^2\)
email words phrase feature such as “verify account”

Feature Engineering

Feature engineering means constructing input features that make useful patterns easier for a model to learn.

Why it matters for logistic regression
The model is linear in the features it receives. Better features can make a difficult pattern easier to separate.

From Weighted Evidence to Rule Paths

Transition question
What if a prediction depends on a sequence of conditions rather than one weighted sum?

Rule-path intuition
If a suspicious link exists, classify as spam. Otherwise, check whether the sender is known. If the sender is unknown, check whether the email asks for a password.

Decision Tree

Decision Tree
A decision tree is a classifier that predicts by following a sequence of feature tests from a root node to a leaf.
Interal Node
An internal node contains a feature test.
Branch
A branch is an outcome of a feature test.
Leaf
A leaf gives the final prediction.

Important distinction
Weights describe global feature effects; tree paths explain individual predictions through tested conditions.

Choosing a Good Split

Tree Learning Problem
A tree must learn which features to test first.

A useful split separates the data into groups that are more class-pure.

Entropy as Label Mixture

Entropy measures how mixed or uncertain the class labels are in a group.

Group label mix Entropy intuition
all spam low entropy
all not spam low entropy
half spam, half not spam high entropy
mostly one class medium entropy

Binary entropy
\[ H(S) = -p_+\log_2p_+-p_-\log_2p_- \]

Information Gain Intuition

Information Gain
Information gain measures how much a split reduces label uncertainty.

\[ \rm information \ gain = entropy \ before \ split - weighted \ entropy \ after \ split \]

Tree Overfitting and Pruning

Pruning
Pruning reduces a decision tree by removing branches that are likely to capture noise rather than useful structure.

Pruning intuition
Remove overly specific branches if they do not improve validation performance.

Comparing the Two Model Families

Aspect Logistic regression Decision tree
Main idea weighted evidence hierarchical rules
Prediction score → probability root-to-leaf path
Geometry linear boundary in feature space axis-aligned splits
Interpretation weights path of tests
Interactions needs engineered features can represent interactions through paths
Overfitting risk many features / large weights deep branches
Scaling often important less sensitive to monotonic scaling

Logistic regression explains through weights. Decision trees explain through paths.
