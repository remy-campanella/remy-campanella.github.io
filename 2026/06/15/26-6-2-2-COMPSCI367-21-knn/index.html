【COMPSCI 367】 Lecture 21 Instance-Based and Probabilistic Classifiers

发表于2026-06-15|更新于2026-06-15|技术学习COMPSCI 367 Artificial Intelligence
|浏览量:

Two Ways to Classify a New Example

A new example arrives. We must assign it to one of several classes.

Reasoning style Question asked by the classifier
Similarity-based Does this example look close to past examples of a class?
Probabilistic Which class best explains the observed features?

A learning algorithm is not only a calculation. It encodes an assumption about what evidence should matter.

The Hidden Assumption Behind a Classifier

A classifier is a supervised learning model that maps an input feature vector to a discrete class label.

Model family Hidden assumption
kNN Class membership is local: nearby examples are informative.
Naive Bayes Class membership explains the probability of observed features.

Inductive Bias: Why Algorithms Generalise Differently

An inductive bias is the set of assumptions that allows a learning algorithm to generalise beyond the training examples.

Why a bias is necessary

  • Training data is finite.
  • New examples may not exactly match old examples.
  • The algorithm must decide what patterns should carry over.
Algorithm Bias
kNN local neighbourhoods matter
Naive Bayes feature evidence can be separated
Decision tree useful rules split features
Linear model weighted feature combinations matter

No inductive bias means no principled generalisation.

kNN: Classification by Local Similarity

k-nearest neighbours classifies a new example by finding the \(k\) closest training examples and predicting by majority vote among their labels.

kNN assumption
Nearby examples in feature space should often behave similarly.

Similarity is not automatic. It depends on the features, distance function, and scaling.

Instance-based learning stores training examples and uses them directly when mak￾ing predictions.

Lazy learning delays most computation until prediction time rather than building a compact model during training.

A distance function measures how far apart two feature vectors are.

Distance is not objective. It depends on which features we choose and how they are scaled.

Feature Scaling Can Change the Answer

Feature scaling transforms features so that their numerical ranges are comparable before distance is computed.

A decision boundary is the boundary in feature space where a classifier changes its predicted class.

kNN choice Boundary shape Link to model complexity
\(k = 1\) jagged, local, sensitive can overfit
moderate \(k\) smoother but still local often better balance
very large \(k\) overly smooth can underfit

Naive Bayes Recap from Lecture 15

Naive Bayes compares class scores of the form
\[ \text{score}(c) = P(c) \prod_i P(x_i | c), \hat{c} = \arg \max_c \text{score}(c) \]

Core assumption
Features are treated as conditionally independent given the class.

Today’s focus
What kind of modelling assumption it makes.

Naive Bayes asks
Which class makes this combination of features most probable?

kNN view
Look for nearby stored examples.

Naive Bayes view
Compare how well each class explains the observed evidence.

Same Data, Different Classifier Logic

Question kNN Naive Bayes
What evidence matters? nearby stored examples priors and likelihoods
How is evidence combined? majority vote multiplication of evidence terms
What can dominate? distance geometry high likelihood ratios
Typical output class vote class score

Same data, different assumption, possibly different prediction.

Failure Modes: Geometry versus Probability

Situation kNN failure mode Naive Bayes failure mode
one feature has huge scale distance is distorted less direct issue
many irrelevant features nearest neighbours become noisy weak likelihoods may dilute scores
correlated text features not the main issue evidence may be double-counted
small local class island can be captured if \(k\) is small may be missed by global likelihoods
high-dimensional sparse text distances may become less meaningful often works well as a baseline
limited training data local neighbourhoods may be unreliable probability estimates may be unstable

Model choice should match the structure we believe the task has.

Strengths and Weaknesses

Aspect kNN Naive Bayes
Training store examples estimate priors and likelihoods
Prediction compute distances and vote compute class scores
Representation geometric feature space probability tables
Main strength flexible local decisions compact, fast, strong for sparse features
Main weakness scaling and irrelevant features independence assumption
Good fit when local neighbourhoods are meaningful features are sparse and evidence-like

kNN trusts neighbourhoods. Naive Bayes trusts decomposed probabilistic evidence.
