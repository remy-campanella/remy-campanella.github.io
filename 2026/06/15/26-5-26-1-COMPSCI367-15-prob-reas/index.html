【COMPSCI 367】 Lecture 15 Probablistic Reasoning—Native Bayes and Bayesian Networks

发表于2026-06-14|更新于2026-06-16|技术学习COMPSCI 367 Artificial Intelligence

From Facts to Degrees of Belief

A degree of belief is a number in \([0, 1]\) representing how strongly an agent believes that a proposition is true.

Probability does not replace logic. It extends reasoning to cases where the truth is hidden, noisy, or only partially observed.

Probability Recap

  1. Conditional Probability
    \[ P(A|B) = \frac{P(A, B)}{P(B)} \]

  2. Product Rule
    \[ P(A, B) = P(A|B)P(B) \]

  3. Marginalisation
    \[ P(A) = \sum_bP(A|b)P(b) \]

  4. Bayes’ Rule
    \[ P(H|E) = \frac{P(E|H)P(H)}{P(E)} \]

Bayes’ Rule as Belief Updating

Bayes’ rule updates belief in a hypothesis \(H\) after observing evidence \(E\):
\[ P(H|E) = \frac{P(E|H)P(H)}{P(E)} \]

Term Meaning
\(P(H)\) prior belief
\(P(E | H)\) likelihood
\(P(E)\) normalising evidence term
\(P(H | E)\) posterior belief

Worked Example: A Diagnostic Test

Setup
A test for an infection has the following probabilities:
\[ P(\text{Inf}) = 0.04, P(\text{Test} | \text{Inf}) = 0.95, P(\text{Test} | \neg \text{Inf}) = 0.01. \]

Question
A patient tests positive. How much should the AI system believe that the patient is infected?

\[ \begin{align} P(\text{Inf} | \text{Test}) &= \frac{P(\text{Test} | \text{Inf})P(\text{Inf})}{P(\text{Test})} \\ &= \frac{P(\text{Test} | \text{Inf})P(\text{Inf})}{P(\text{Test} | \text{Inf})P(\text{Inf}) + P(\text{Test} | \neg \text{Inf})P(\neg \text{Inf})} \\ &= \frac{0.95 \times 0.04}{0.95 \times 0.04 + 0.01 \times 0.96} \\ & \approx 0.7983 \end{align} \]

The Base-Rate Trap

\[ P(\text{Disease} | +) \neq P(+ | \text{Disease}) \]

A positive test does not automatically imply a high disease probability. The posterior depends on both test accuracy and the base rate.

From Belief Updating to Classification

Classification problem
A classifier receives observed features and chooses a class.

Bayesian view
\[ \hat{c} = \arg \max_{c \in C}P(c | x_1, \ldots, x_n) \]

Naive Bayes Classifier

A Naive Bayes classifier predicts the class \(c\) that maximises
\[ \mathbf{Score}(c) = P(c)\prod_{i = 1}^nP(x_i | c) \]

where \(x_1, \ldots, x_n\) are observed features.

Features \(X_1, \ldots, X_n\) are conditionally independent given class \(C\) if, once \(C\) is known,
\[ P(X_1, \ldots, X_n | C) = \prod_{i = 1}^nP(X_i | C) \]

From Naive Bayes to Bayesian Networks

Naive Bayes assumption
All features are conditionally independent once the class is known.

A Bayesian network generalises Naive Bayes by representing variables as nodes and direct probabilistic dependencies as directed edges.

Bayesian Networks

Bayesian Network
A Bayesian network is a directed acyclic graph whose nodes are random variables and whose edges represent direct probabilistic dependencies in the model.
Parent
The parents of a node \(X\), written \(\text{Pa(X)}\), are the nodes with directed edges into \(X\).
Local Probability Model
Each node \(X\) is equipped with a local conditional probability distribution.
\[ P(X | \text{Pa}(X)) \]

Weather Network
Let
\[ C = \text{Cloudy}, S = \text{Sprinkler}, R = \text{Rain}, W = \text{WetGrass} \]

Parent Sets
\[ \text{Pa}(C) = \varnothing, \text{Pa}(S) = \{C\}, \text{Pa}(R) = \{C\}, \text{Pa}(S) = \{R, W\} \]

Conditional Probability Tables

A conditional probability table lists the probability of a node for each possible assignment of its parents.

CPTs for the weather network
\[ P(C), P(S | C), P(R | C), P(W | S, R) \]

Factorising a Joint Distribution

A Bayesian network factorises a joint distribution as
\[ P(X_1, \ldots, X_n) = \prod_{i = 1}^nP(X_i | \text{Pa}(X_i)) \]

The graph tells us which local probability tables are needed to describe the joint distribution.

Why factorisation saves space
Conditional independence lets the model store local dependencies instead of every complete world state.

Inference by Enumeration

Infeence Enumeration
Inference by enumeration computes a query probability by summing over unobserved variables and then normalising.

Example query
\[ P(R | W = \mathsf{true}) \]

Here \(R\) is the query variable and \(W = \mathsf{true}\) is observed evidence.

The variables \(C\) and \(S\) are not observed, so we sum over their possible values.

For each possible values \(r = \{R, \neg R\}\), define
\[ \text{score} = \sum_c \sum_s P(c)P(s|c)P(r|c)P(w|s, r) \]

where \(w\) means \(W = \mathsf{true}\).

Normalise the two scores,
\[ P(r|w) = \frac{\text{score}(r)}{\text{score}(R) + \text{score}(\neg R)} \]

Enumeration prodedure

Enumeration is exact, but it becomes expensive when there are many hidden variables.
