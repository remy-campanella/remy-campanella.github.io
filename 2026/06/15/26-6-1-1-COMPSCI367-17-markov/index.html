【COMPSCI 367】 Lecture 17 Markov Decision Process(MDP) and Policy Evaluation

发表于2026-06-14|更新于2026-06-14|技术学习COMPSCI 367 Artificial Intelligence
|浏览量:

Sequential Decision Problem

A sequential decision problem is one in which an agent repeatedly chooses actions, observes consequences, receives rewards, and continues from the resulting state.

MDP = modelling languange for sequential decision problem.

Grid-World Running Example

A robot moves in a small grid-world. Some cells are terminal, one cell is blocked, and each ordinary move has a small cost.

Markov Decision Processes

A Markov decision process is a tuple
\[ M = (\mathcal{S}, \mathcal{A}, P, R, \gamma), \]

where \(\mathcal{S}\) is a set of states, \(\mathcal{A}\) is a set of actions, \(P(s' | s, a)\) gives transition probabilities, \(R(s, a, s′)\) gives immediate rewards, and \(γ \in [0, 1)\) is a discount factor.

Component Meaning in grid-world
\(\mathcal{S}\) possible grid cells
\(\mathcal{A}\) movement actions
\(P(s' | s, a)\) probability that action \(a\) in state \(s\) leads to \(s'\)
\(R(s, a, s′)\) reward received on transition \(s \rightarrow s'\)
\(\gamma\) how much future rewards count

The Markov Property

The Markov property says that, given the current state and action, the distribution of the next state and reward does not depend on the earlier history:
\[ P(s_{t + 1}, r_{t + 1} | s_t, a_t, s_{t - 1}, a_{t - 1}, \ldots) = P(s_{t + 1}, r_{t + 1} | s_t, a_t) \]

A good state description contains all information needed for predicting the next step.

Policies

Policy
A policy tells the agent what action to take in each state. A deterministic policy has the form
\[ \pi: \mathcal{S} \rightarrow \mathcal{A} \]

A stochastic policy gives action probabilities \(\pi(a | s)\).

In a deterministic grid-world policy, each non-terminal cell can be labelled with an arrow. The arrow is the intended action chosen at that state.

The policy chooses an intended action. The transition model determines what may actually happen next.

Reward, Return, and Discounting

Reward
A reward is the immediate numerical feedback received after a transition.
Return
A return is the total reward accumulated over time.

Discounted return from time 0
\[ G_0 = r_1 + \gamma r_2 + \gamma^2 r_3 + \ldots \]

Reward is local and immediate. Return is a whole future path measured from the current time.

Discount Factor
The discount factor \(\gamma \in [0, 1)\) controls how much future rewards count compared with immediate rewards.
Value of \(\gamma\) Interpretation
close to 0 the agent mainly cares about immediate rewards
close to 1 the agent cares more about long-term rewards

State-Value Function

The state-value function \(V^\pi(s)\) is the expected discounted return starting from state \(s\) and then following policy \(\pi\):
\[ V^\pi(s) = \mathbb{E}_\pi \left[\sum_{t = 0}^\infty \gamma^t r_{t + 1} | s_0 = s\right] \]

\(V^\pi(s)\) asks: if we start here and follow \(\pi\), how good is this state in expectation?

Value is policy-dependent. A different policy can give a different value for each state.

Policy Evaluation

Policy evaluation takes an MDP and a fixed policy \(\pi\), and computes the value function \(V^\pi\).

Policy evaluation does not find the best policy. It evaluates a policy that is already given.

Bellman Expectation Equation

The Bellman expectation equation expresses the value of a state under a fixed policy as immediate reward plus discounted expected successor value.

\[ V^\pi(s) = \sum_{s' \in S} P(s' | s, \pi(s))[R(s, \pi(s), s') + \gamma V^\pi(s')] \]

value now = expected immediate reward + discounted expected successor value.

Define the expected immediate reward under policy \(\pi\):
\[ R^\pi(s) = \sum_{s' \in S} P(s' | s, \pi(s))R(s, \pi(s), s') \]

Then
\[ V^\pi(s) = R^\pi(s) + \gamma \sum_{s' \in S} P(s' | s, \pi(s)) V^\pi(s') \]

One Bellman Backup

Battery Robot Setup
State \(L\) means low battery. The fixed policy says
\[ \pi(L) = recharge. \]

The recharge action moves to high battery \(H\) with probability 0.9, and remains in low battery \(L\) with probability 0.1.

Transition model
\[ P(H | L, \text{recharge}) = 0.9 \\ P(L | L, \text{recharge}) = 0.1 \]

Values and rewards
\[ V_{\rm old}(H) = 8, V_{\rm old} = 4 \\ R(L, \text{recharge}, H) = 0, R(L, \text{recharge},L) = 0 \\ \gamma = 0.9 \]

Backup calculation
\[ \begin{align} V_{\rm new}(L) &= 0.9(0 + 0.9 \times 8) + 0.1(0 + 0.9 \times 4) \\ &= 6.84 \end{align} \]

The value of \(L\) increases because recharging is likely to move the robot to high battery, which has high future value.

Iterative Policy Evaluation

Iterative policy evaluation repeatedly applies Bellman expectation updates until the value function stabilises.

This is a form of bootstrapping: new estimates are computed using old estimates.

Iterative Beellman Update
\[ V_{k + 1}^\pi(s) = \sum_{s' \in S}P(s' | s, \pi(s))[R(s, \pi(s), s') + \gamma V_k^\pi(s')] \]

Convergence idea
Repeated updates converge to the value function of the fixed policy:
\[ V^\pi(s) = \lim_{k \to \infty} V_k^\pi(s) \]
