【COMPSCI 367】 Lecture 8 Population-Based and Stochastic Search

发表于2026-06-12|更新于2026-06-15|技术学习COMPSCI 367 Artificial Intelligence

The Central Question

Last lecture, we saw that greedy local search keeps one complete candidate solution and repeatedly moves to a better neighbour.

But sometimes reaching a better region requires first moving to a worse solution.

  • Can we sometimes accept a worse move?
  • Can we keep more than one candidate?
  • Can we combine useful parts of different solutions through variation?

Simulated Annealing

In metallurgy, annealing is a heat-treatment process. A metal is heated so that atoms can move and internal stresses can relax. It is then cooled slowly, allowing the material to settle into a more stable, low-energy structure.

At high temperature, the search is willing to “shake” itself out of shallow local optima. As the temperature cools, the search becomes increasingly selective.

Suppose a move makes the solution worse by
\[ \Delta = C(y) - C(x) > 0 \]

we want the acceptance probability that satisfies

  • Larger \(\Delta\) should give smaller acceptance probability.
  • High \(T\) should make the search more willing to explore; low \(T\) should make it more selective.
  • A worse move should not be rejected automatically; it should have a graded chance of being accepted.

A natural choice for soft probabilistic decision is the Boltzmann factor:
\[ P(\text{accept worse move}) = e^{\frac{-\Delta}{T}} \]

Simulated Annealing Rule
Suppose the current solution is \(x\), a randomly generated neighbour is \(y\), and we minimise cost \(C\).

Define the cost difference
\[ \Delta = C(y) - C(x) \]

  • Improving or equal: If \(\Delta \leq 0\), accept \(y\).
  • Worsening: If \(\Delta > 0\), accept \(y\) with probability \(P = e^{\frac{-\Delta}{T}}\).

The annealing schedule changes the temperature overtime. A common schedule is geometric cooling
\[ T_t = \alpha_t T_0, 0 < \alpha < 1 \]

Tuning issue
Cooling too fast makes the search greedy too early; cooling too slowly may waste computation.

Stimulated Annealing Pseudocode

Because annealing may accept worse moves, we usually return the best solution seen, not necessarily the final solution.

Beam Search

Keeping only one candidate is often too aggressive. A single greedy choice may discard a path that looks slightly worse now but could lead to a much better solution later.

Beam search keeps the best \(k\) candidates at each step, allowing more diversity in the search.

Beam search pseudocode

  • \(k = 1\): Greedy search.
  • Larger \(k\): More exploration and memory.

Risk: Early pruning may delete the fututr winner.

Version Candidate type Typical use
Beam search Partial solutions. Sequence generation, planning, constructive search.
Local beam search Complete solutions. Local optimisation over complete assignments.

Instead of deterministically keeping the top \(M\), stochastic beam search samples candidates with probability biased toward low cost:
\[ P(y) = \frac{e^{-\beta C(y)}}{\sum_ze^{-\beta C(z)}} \]

Increasing \(\beta\) increases selection pressure toward the best candidate.

Genetic Algorithms

Genetic algorithms: A population of complete candidate solutions evolve over time through selection, crossover, mutation, and replacement.

New candidates are created by combining and modifying old ones.

graph LR

P("Population many candidates")
E("Evaluate compute fitness")
S("Select prefer fitter candidates")
C("Combine create offsprings")
M("Mutate small random changes")
R("Replace from next generation")

P --> E --> S --> C --> M --> R --> P

Genetic Algorithm Pseudocode

Selection refers to choosing candidate to produce offsprings. It should give better candidates a higher chance of becoming parents.

Example: Crossover Can Combine Building Blocks
Target string:
\[ \tau = 10110110 \]

Fitness is the number of matching positions with \(\tau\).

Parents:
\[ p_1 = 10110000, p_2 = 00000110 \]

Their fitness values are
\[ F(p1) = 6, F(p2) = 5. \]

Perform one-point crossover after position 5:
\[ p_1 = 10110 | 000, p_2 = 00000 | 110 \]

Children:
\[ c_1 = 10110 | 110 = 10110110, c_2 = 00000 | 000 = 00000000 \]

\(F(c_1) = 8\) is excellent, but \(F(c_2) = 3\) is poor.

Exercise: Genetic Algorithm for 8-Queens

Cost and Fitness
Two queens attack each other if
\[ r_i = r_j \text{ or } |r_i - r_j| = |i - j| \]

Define
\[ C(s) = \sum_{1 \leq i \leq j \leq 8} \mathbf{1}[r_i = r_j \text{ or } |r_i - r_j| = |i - j|] \]

A simple fitness function
\[ F(s) = \frac{1}{1 + C(s)} \]

A valid solution is \(C(s) = 0, F(s) = 1\).

Mutation
Choose one column and change its row.

Crossover
Combine prefixes and suffixes from two paents.
\[ (1, 3, 5, 3 | 8, 2, 4, 7) \times (2, 5, 7, 4 | 1, 3, 6, 8) \to (1, 3, 5, 3 | 1, 3, 6, 8) \]

Selection
Candidates with higher fitness should be more likely to become parents.

For example, using fitness-proportionate selection
\[ P(s_i) = \frac{F(s_i)}{\sum_jF(s_j)} \]

Replacement
After crossover and mutation, the algorithm forms the next generation. A common choice is to keep a few best candidates and fill the rest with offspring.

Important modelling point
The representation
\[ s = (r_1, r_2, \ldots, r_8) \]
guarantees exactly one queen in each column. But it does not prevent row or diagonal conflicts. Those are handled by the cost/fitness function.

Unified Comparison

Method Keeps Main parameter Main risk
Hill climbing 1 solution neighbourhood local optimum
Simulated annealing 1 solution temperature schedule slow convergence or premature greediness
Beam search \(k\) candidates beam width pruning good paths
Genetic algorithm population population size, \(p_c\), \(p_m\) premature convergence or poor representation

These methods are not universally better or worse. They make different trade-offs between memory, randomness, diversity, and computation.
