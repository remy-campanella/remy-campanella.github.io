Recall: A RescueBot World

Propositional repetition
Suppose the robot may enter any room that is safe and connected to its current room.

With propositional atoms, we may need many separate rules:
\[ SafeRoom21 \land ConnectedRoom11Room21 \to CanEnterRoom21,\\ SafeRoom12 \land ConnectedRoom11Room12 \to CanEnterRoom12,\\ SafeRoom22 \land ConnectedRoom11Room22 \to CanEnterRoom22. \]

Problem
The pattern is the same, but the room names keep changing.

Desired representation
One reusable rule should describe the general pattern.

Objects and Constants

Object
An object is something in the world that the knowledge base talks about.
Constant
A constant is a name for one specific object.

First-order logic lets us build structured statements from names of objects and predicates.

\[ CanEnterRescueBotRoom21 \Rightarrow CanEnter(RescueBot, Room21) \]

Predicates and Atomic Formulas

Predicate
A predicate represents a property of objects or a relation among objects.
Atomic Formula
An atomic formula is a predicate applied to suitable arguments.

In rescue bot example,

  • Properties: \(Robot(RescueBot)\), \(Room(Room21)\), \(Safe(Room21)\).
  • Relations: \(Connected(Room11, Room21)\), \(At(RescueBot, Room21)\).

Arity
The arity of a predicate is the number of arguments it takes.

Universal Quantification

Universal Quantification
Universal quantification uses \(\forall\) to say that a formula holds for every object in the domain.

\[ \forall x (Safe(x)\rightarrow Room(x)) \]

Read: For every object \(x\), if \(x\) is safe, then \(x\) is a room.

A universal implication is violated only by an object that makes the condition true and the conclusion false.

Wrong Version
\[ \forall x (A(x) \land B(x)) \]

Existential Quantification

Existential Quantification
Existential quantification uses \(\exists\) to say that at least one object in the domain makes the formula true.

The pattern Some \(A\) are \(B\) is often written as
\[ \exists x (A(x) \land B(x)) \]

Witness
A witness for an existential formula is an object that makes the formula true.

Common Wrong Version
\[ \exists x(Room(x) \to Safe(x)) \]

This can be true because of an object that is not a room. It does not express “some room is safe”.

Translation Patterns

English FOL Visual
All \(A\) are \(B\). \(\forall x(A(x) \rightarrow B(x))\) Check every \(A\), \(B\) must hold.
Some \(A\) are \(B\). \(\exists x(A(x) \land B(x))\) Find one \(A\) that is also \(B\).
No \(A\) are \(B\). \(\forall x(A(x) \rightarrow \neg B(x))\) Check every \(A\), \(B\) must not hold.
Some \(A\) are not \(B\) \(\exists x(A(x) \land \neg B(x))\) Find one \(A\) that is not \(B\).

Quantifier Scope

Quantifier Scope
The scope of a quantifier is the part of formula that the quantifier controls.

Always use parentheses to make the scope clear.

Quantifier Order

Translate:

Every robot has a safe room it can enter.

to:
\[ \forall r(Robot(r) \rightarrow \exists y(Room(y) \land Safe(y) \land CanEnter(r, y))) \]

FOL Inference

For every robot \(r\) and rooms \(x\), \(y\), if \(r\) is at \(x\), \(x\) is connected to \(y\), and \(y\) is safe, then \(r\) can enter \(y\).
\[ \forall r \forall x \forall y((Robot(r) \land At(r, x) \land Connected(x, y) \land Safe(y)) \rightarrow CanEnter(r, y)) \]

Universal rules use implication; existential claims use conjunction.
