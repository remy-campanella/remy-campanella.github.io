From Policy Evaluation to Optimal Control

A good action should consider immediate reward plus discounted future value.

What We Assume in This Lecture

In this lecture, we assume the agent has a known model of the environment: it knows the transition probabilities \(P(s' | s, a)\) and rewards \(R(s, a, s')\).

One-Step Lookahead

Given a value estimate \(V\) , the one-step lookahead value of taking action a in state \(s\) is
\[ Q_V(s, a) = \sum_{s' \in S} P(s' | s, a)[R(s, a, s') + \gamma V(s')] \]

\(Q_V(s, a)\) = reward now + discounted expected value later.

State Values and Action Values

Given a value estimate \(V\) , a greedy action at state \(s\) is an action with the largest one-step lookahead value:
\[ \pi_{\text{greedy}(s)} \in \arg \max_{a \in A} Q_V(s, a) \]

Try each action using one-step lookahead. Choose the action with the largest estimated return.

Greedy here means greedy with respect to future-aware lookahead value, not just immediate reward.

Optimal Value

The optimal value function \(V^*(s)\) is the maximum expected discounted return achievable from state \(s\), when the agent is free to choose actions optimally.
\[ V^*(s) = \max_\pi V^\pi(s) \]

i.e. best value we can achieve from \(s\).

Bellman Optimality

The Bellman optimality equation says that \(V^*\) satisfies
\[ V^*(s) = \max_{a} \sum_{s' \in S} P(s' | s, a)[R(s, a, s') + \gamma V^*(s')] \]

If \(V^*(s')\) tells us the best future value from every next state \(s'\), then from current state \(s\):

  • try each action \(a\);
  • compute expected reward plus discounted optimal future value;
  • choose the best action.

Best value now = best one-step decision now + best future value later.

From Bellman Optimality to Value Iteration

The Bellman optimality equation contains \(V^*\) on both sides. We usually do not know \(V^*\) in advance.

Start with a rough value estimate \(V_0\), then repeatedly improve it:
\[ V_0 \rightarrow V_1 \rightarrow V_2 \rightarrow \ldots \rightarrow V^* \]

Value Iteration

Value iteration repeatedly applies Bellman optimality backups:
\[ V_{k + 1}(s)\max_{a} \sum_{s' \in S} P(s' | s, a)[R(s, a, s') + \gamma V_k(s')] \]

Stopping Value Iteration

In practice, stop when the value table changes very little:
\[ \Delta_k = \max_s |V_{k + 1} - V_k(s)| \]

Stop when
\[ \Delta_k < \varepsilon \]

Stopping early gives an approximate value function. The smaller \(\varepsilon\) is, the more accurate the value estimate is expected to be.

Extracting a Greedy Policy

After value iteration produces a value table \(V\), choose
\[ \pi(s) \in \arg \max_a \sum_{s'}P(s' | s, a)(R(s, a, s') + \gamma V(s')) \]

For deterministic gridworlds:
\[ \pi(s) \in \arg \max_a (R(s, a, s') + \gamma V(s')) \]

Values tell us which states are promising. A greedy policy chooses actions leading toward promising states.
